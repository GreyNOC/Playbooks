01 / Credential Access

Password Spraying

Low-volume, high-fan-out authentication abuse against identity providers. A spray hides per-user — and only becomes visible when failures are aggregated across the user dimension.

MITRE ATT&CK

TechniqueIDTactic
Brute Force: Password SprayingT1110.003Credential Access
Valid AccountsT1078Initial Access / Persistence

Key Indicators

  • Single source authenticating against many distinct usernames in a short window.
  • Failure-to-user ratio close to 1:1 (each user fails roughly once).
  • Bursts against legacy auth protocols (BAV2ROPC, IMAP, POP, SMTP AUTH).
  • Unusual user-agents (python-requests, MSOLSpray, FireProx).
  • Successful login from the same source shortly after the fan-out — highest-value signal.

Sample Detection Logic

// Distinct-user fan-out from a single source within a sliding window.
{
  "rule_name": "Password Spray - Distinct User Fan-Out",
  "data_source": ["AzureAD.SignInLogs", "Okta.SystemLog", "ADFS.SecurityLog"],
  "logic": {
    "window": "10m",
    "group_by": ["src_ip", "user_agent"],
    "where": {
      "event_type": "authentication",
      "result": "failure",
      "failure_reason_in": ["invalid_password", "50126"]
    },
    "having": {
      "distinct_users_gte": 15,
      "avg_failures_per_user_lte": 3
    }
  },
  "severity": "high",
  "tags": ["T1110.003", "credential_access", "identity"]
}

02 / Credential Access

Brute Force Attack

High-volume password guessing against a single account or service. Noisy but effective against weak credentials, exposed services, and accounts without lockout — and still one of the most common access vectors for SSH, RDP, and VPN concentrators.

MITRE ATT&CK

TechniqueIDTactic
Brute ForceT1110Credential Access
Password GuessingT1110.001Credential Access
External Remote ServicesT1133Initial Access

Key Indicators

  • ≥ N authentication failures against a single user/host within a short window.
  • Failures followed by a success from the same source — treat as confirmed compromise.
  • Repeated SYN to authentication ports (22, 3389, 445, 1433, 3306, 5432).
  • Sequential username enumeration patterns.
  • Default-credential pairs (admin/admin, root/toor, sa/sa).

Sample Detection Logic

// Sustained per-target failures with post-success enrichment.
{
  "rule_name": "Brute Force - Single Target High Failure Volume",
  "data_source": ["Linux.Auth", "WindowsEvent.Security", "VPN.Auth", "Firewall.Conn"],
  "logic": {
    "window": "5m",
    "group_by": ["src_ip", "dst_user"],
    "where": { "event_type": "authentication", "result": "failure" },
    "having": { "failure_count_gte": 25 },
    "enrich_with_post_success": true
  },
  "severity": "high",
  "tags": ["T1110", "credential_access", "perimeter"]
}

03 / Reconnaissance

Distributed Port Scan

Multi-source coordinated reconnaissance that evades per-IP thresholds. Each source touches few ports; the aggregate coverage of your assets is comprehensive. Detection has to pivot to the destination dimension and correlate sources by timing and fingerprint.

MITRE ATT&CK

TechniqueIDTactic
Network Service DiscoveryT1046Discovery
Active Scanning: IP BlocksT1595.001Reconnaissance
Active Scanning: VulnerabilityT1595.002Reconnaissance

Key Indicators

  • Many distinct sources probing one destination in a short window.
  • Sequential or strided port coverage across sources.
  • Sources sharing TCP fingerprint (TTL, window size) or JA3/JA4 hashes.
  • SYN without ACK at scale (half-open scan signature).
  • Tightly clustered start times across geographically disparate ASNs.

Sample Detection Logic

// Many sources, one destination, broad port coverage in a narrow window.
{
  "rule_name": "Distributed Port Scan - Many Sources, One Destination",
  "data_source": ["Firewall.Conn", "NetFlow", "IDS.Alerts"],
  "logic": {
    "window": "10m",
    "group_by": ["dst_ip"],
    "where": { "tcp_flags_in": ["SYN"], "bytes_in_lt": 200 },
    "having": {
      "distinct_src_ip_gte": 25,
      "distinct_dst_port_gte": 30,
      "avg_ports_per_src_lte": 5
    }
  },
  "severity": "medium",
  "tags": ["T1046", "T1595", "reconnaissance", "perimeter"]
}
